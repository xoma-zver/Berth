# Berth

Докинг-библиотека для [Avalonia](https://avaloniaui.net) по модели IntelliJ IDEA: шесть фиксированных слотов тул-панелей (3 стороны × 2 группы), пять режимов отображения (dock pinned / dock unpinned / undock / float / window), документная зона как дерево сплитов с таб-группами, сохранение и восстановление раскладки. Сознательно **не** free-form докинг: строгие правила → перечислимое пространство состояний → надёжность.

Целевые платформы: десктоп (Windows/macOS/Linux) и браузер (WASM; плавающие панели — оверлеями). Мобильные хосты собираются best effort, поведение докинга на таче не специфицируется.

**Статус: фазы 1 (ядро tool windows без UI, задачи 1.1–1.8) и 2 (статический UI + ходячий скелет, задачи 2.1–2.4) завершены.** В ядре: типы состояния и инварианты, операции с вытеснением слоёв, веса и ресайзы, дерево таб-групп док-зоны с нормализацией и инвариантами, операции док-зоны с активностью и фолбэками, персистентность (JSON-формат со схемой, `Apply` с отчётом об исправлениях, golden-тесты), жизненный цикл контента (политики, фабрики, заявочное владение вкладками, координатор `ContentLifecycle`) и деревья внутри панелей — `ContentTree`, canHost, вкладка-тело с мостом фабрик, перемещения панель ↔ док-зона теми же командами (`Berth.Core` + тесты). В UI-слое: `BerthWorkspace` материализует состояние — стрипы с «⋯», панели сторон со сплиттерами, декоратор панели, Undock-оверлей — обрабатывает ввод командами ядра (клик по иконке, меню, флайаут «⋯», drag сплиттеров с коммитом на отпускании) и связан с жизненным циклом контента: тело панели материализуется через `ContentLifecycle` (Control — напрямую, view model — через DataTemplates приложения), каждый жест докладывается `NotifyTransition` (`Berth.Avalonia` + headless-тесты). Демо-скелет живёт в `samples/Berth.Demo` (панели с разными политиками жизненного цикла, MVVM- и Control-контент) и собирается всегда, включая CI. Геометрия пары и Undock-оверлея выверена по трассировкам эталона после обкатки демо: доля пары выводится нормализацией предпочтений панелей, толщина оверлея — вес стороны. Обе поведенческие спеки приняты (tool-windows v0.12, document-area v0.8); на очереди фаза 3 — режимы и фокус (автоскрытие, шорткаты).

- Пакеты: `Berth.Core` (модель, без зависимостей на UI) и `Berth.Avalonia` (контролы); пространства имён `Berth` / `Berth.Controls`
- Документация и спецификации: [docs/](docs/README.md)
- Поведенческие спеки: [tool-windows](docs/spec/tool-windows.md), [document-area](docs/spec/document-area.md)
- План работ: [docs/BACKLOG.md](docs/BACKLOG.md)

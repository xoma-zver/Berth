# Berth

Докинг-библиотека для [Avalonia](https://avaloniaui.net) по модели IntelliJ IDEA: шесть фиксированных слотов тул-панелей (3 стороны × 2 группы), пять режимов отображения (dock pinned / dock unpinned / undock / float / window), документная зона как дерево сплитов с таб-группами, сохранение и восстановление раскладки. Сознательно **не** free-form докинг: строгие правила → перечислимое пространство состояний → надёжность.

Целевые платформы: десктоп (Windows/macOS/Linux) и браузер (WASM; плавающие панели — оверлеями). Мобильные хосты собираются best effort, поведение докинга на таче не специфицируется.

**Статус: фазы 1 (ядро tool windows без UI, задачи 1.1–1.8) и 2 (статический UI + ходячий скелет, задачи 2.1–2.4) завершены.** В ядре: типы состояния и инварианты, операции с вытеснением слоёв, веса и ресайзы, дерево таб-групп док-зоны с нормализацией и инвариантами, операции док-зоны с активностью и фолбэками, персистентность (JSON-формат со схемой, `Apply` с отчётом об исправлениях, golden-тесты), жизненный цикл контента (политики, фабрики, заявочное владение вкладками, координатор `ContentLifecycle`) и деревья внутри панелей — `ContentTree`, canHost, вкладка-тело с мостом фабрик, перемещения панель ↔ док-зона теми же командами (`Berth.Core` + тесты). В UI-слое: `BerthWorkspace` материализует состояние — стрипы с «⋯», панели сторон со сплиттерами, декоратор панели, Undock-оверлей — обрабатывает ввод командами ядра (клик по иконке, меню, флайаут «⋯», drag сплиттеров с коммитом на отпускании) и связан с жизненным циклом контента: тело панели материализуется через `ContentLifecycle` (Control — напрямую, view model — через DataTemplates приложения), каждый жест докладывается `NotifyTransition` (`Berth.Avalonia` + headless-тесты). Демо-скелет живёт в `samples/Berth.Demo` (панели с разными политиками жизненного цикла, MVVM- и Control-контент) и собирается всегда, включая CI. Геометрия пары и Undock-оверлея выверена по трассировкам эталона после обкатки демо: доля пары выводится нормализацией предпочтений панелей, толщина оверлея — вес стороны. Фаза 3 (режимы и фокус) завершена: материализация `BerthWorkspace` стала инкрементальной (TW-9.13, задача 3.0) — постоянный каркас с коллапсом ячеек, кэш хостов панелей, diff-обновление на смену состояния; клавиатурный фокус и view-state контента не теряются как побочный эффект материализации, вид контента строится однократно и переживает перекладку и закрытие/переоткрытие. Поверх неё заработали автоскрытие и активация (TW-6.1/6.2/6.6, задача 3.1): DockUnpinned/Undock-панель закрывается, потеряв клавиатурный фокус (попапы, деактивация приложения и меню собственной иконки — исключения) или по клику вне её; активация командой переносит фокус в контент панели, получение фокуса контентом активирует панель, активная подсвечена. Фазу закрыли шорткаты активации (TW-5.5, задача 3.2): приложение привязывает свой кеймап к публичной команде `BerthWorkspace.ActivateToolWindow` с трёхпозиционной логикой («активна» = клавиатурный фокус внутри панели, как у эталона), тултип иконки стрипа показывает подсказку шортката от необязательного провайдера приложения, а перенос фокуса TW-6.6 расширен реатачем хоста активной панели — перенос в другой слот или слой восстанавливает фокус, не давая автоскрытию закрыть перемещаемую панель. Фаза 4 (документная зона UI) начата: задача 4.0 материализовала дерево док-зоны главного окна — таб-группы с полосами вкладок, сплиты со сплиттерами, контент активной вкладки каждой группы; хосты вкладок кэшируются по id и переприсоединяются только семантикой команды (DA-9.6 — зеркало TW-9.13: ключёванная реконсиляция, дети сплита размещаются Grid-индексами, поворот сплита не реатачит вовсе), контент тянется лениво планировщиком вне рендера (отказ фабрики закрывает вкладку штатно, спящие вкладки — заглушки с заголовком от публичного `TabTitleProvider`), ввод — клики и меню вкладок (Close, Split×4, Rotate, Move to панели-владельца) и сплиттеры → `SetSplitShares`; проводка активности замкнута: фокус в контенте вкладки активирует её и сбрасывает подсветку панелей (DA-6.4, TW-6.5), `Esc` из панели ведёт в текущую вкладку (TW-6.3), закрытие сфокусированной панели возвращает фокус документу (DA-E21). Окна документов — фаза 6. Обе поведенческие спеки приняты (tool-windows v0.16, document-area v0.9); следующая задача — полоса вкладок внутри панелей (4.1).

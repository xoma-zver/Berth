# Berth

Докинг-библиотека для [Avalonia](https://avaloniaui.net) по модели IntelliJ IDEA: шесть фиксированных слотов тул-панелей (3 стороны × 2 группы), пять режимов отображения (dock pinned / dock unpinned / undock / float / window), документная зона как дерево сплитов с таб-группами, сохранение и восстановление раскладки. Сознательно **не** free-form докинг: строгие правила → перечислимое пространство состояний → надёжность.

Целевые платформы: десктоп (Windows/macOS/Linux) и браузер (WASM; плавающие панели — оверлеями). Мобильные хосты собираются best effort, поведение докинга на таче не специфицируется.

**Статус: фаза 1 (ядро tool windows без UI) в процессе.** Закрыты задачи 1.1–1.3: типы состояния и инварианты, операции ядра с вытеснением слоёв, веса и ресайзы (`Berth.Core` + тесты). Обе поведенческие спеки приняты (tool-windows v0.5, document-area v0.3); на очереди задача 1.5 — дерево таб-групп в ядре.

- Пакеты: `Berth.Core` (модель, без зависимостей на UI) и `Berth.Avalonia` (контролы); пространства имён `Berth` / `Berth.Controls`
- Документация и спецификации: [docs/](docs/README.md)
- Поведенческие спеки: [tool-windows](docs/spec/tool-windows.md), [document-area](docs/spec/document-area.md)
- План работ: [docs/BACKLOG.md](docs/BACKLOG.md)

Решение — `Berth.slnx`; демо-хосты (`Berth.Demo*`) собраны в `samples/`. Папка репозитория пока исторически называется `IdeaDocking` — её переименование опционально (см. бэклог).

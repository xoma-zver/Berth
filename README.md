# Berth

Докинг-библиотека для [Avalonia](https://avaloniaui.net) по модели IntelliJ IDEA: шесть фиксированных слотов тул-панелей (3 стороны × 2 группы), пять режимов отображения (dock pinned / dock unpinned / undock / float / window), документная зона как дерево сплитов с таб-группами, сохранение и восстановление раскладки. Сознательно **не** free-form докинг: строгие правила → перечислимое пространство состояний → надёжность.

Целевые платформы: десктоп (Windows/macOS/Linux) и браузер (WASM; плавающие панели — оверлеями). Мобильные хосты собираются best effort, поведение докинга на таче не специфицируется.

**Статус: фаза 1 (ядро tool windows без UI) завершена — задачи 1.1–1.8; фаза 2 (статический UI + ходячий скелет) в работе — задачи 2.1–2.2 готовы.** В ядре: типы состояния и инварианты, операции с вытеснением слоёв, веса и ресайзы, дерево таб-групп док-зоны с нормализацией и инвариантами, операции док-зоны с активностью и фолбэками, персистентность (JSON-формат со схемой, `Apply` с отчётом об исправлениях, golden-тесты), жизненный цикл контента (политики, фабрики, заявочное владение вкладками, координатор `ContentLifecycle`) и деревья внутри панелей — `ContentTree`, canHost, вкладка-тело с мостом фабрик, перемещения панель ↔ док-зона теми же командами (`Berth.Core` + тесты). В UI-слое: `BerthWorkspace` материализует состояние — стрипы с «⋯», панели сторон со сплиттерами, декоратор панели, Undock-оверлей — и обрабатывает ввод командами ядра: клик по иконке (toggle), меню режимов/перемещений, флайаут «⋯», drag сплиттеров с коммитом на отпускании (`Berth.Avalonia` + headless-тесты). Обе поведенческие спеки приняты (tool-windows v0.11, document-area v0.8); на очереди 2.3 — демо-скелет с мостом контента.

- Пакеты: `Berth.Core` (модель, без зависимостей на UI) и `Berth.Avalonia` (контролы); пространства имён `Berth` / `Berth.Controls`

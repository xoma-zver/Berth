# Berth

Докинг-библиотека для [Avalonia](https://avaloniaui.net) по модели IntelliJ IDEA: шесть фиксированных слотов тул-панелей (3 стороны × 2 группы), пять режимов отображения (dock pinned / dock unpinned / undock / float / window), документная зона как дерево сплитов с таб-группами, сохранение и восстановление раскладки. Сознательно **не** free-form докинг: строгие правила → перечислимое пространство состояний → надёжность.

Целевые платформы: десктоп (Windows/macOS/Linux) и браузер (WASM; плавающие панели — оверлеями-псевдоокнами). Мобильные хосты собираются best effort, поведение докинга на таче не специфицируется.

## Статус

**0.1.x (pre-1.0): функциональность v1 завершена**, публичный API и схема персистентности ещё могут двигаться. Что умеет библиотека:

- **Тул-панели**: шесть слотов, пять режимов с вытеснением по слоям, автоскрытие по фокусу, стрипы с быстрым доступом «⋯», двухуровневые меню, шорткаты активации, геометрия сторон и пар в долях (пиксели в ядро не проникают).
- **Документная зона**: n-арное дерево сплитов с таб-группами и нормализацией к канонической форме, окна документов, единая модель дерева для контента панелей — вкладки переезжают между панелью и док-зоной теми же командами.
- **Плавающий слой**: настоящие окна ОС на десктопе, псевдоокна в оверлее браузера; хранимый режим не деградирует — только эффективное представление (`CanFloat`/`CanUseWindowed`).
- **DnD**: перетаскивание иконок, заголовков и вкладок внутри и между окнами — тонкий слой над командами ядра (каждый жест доступен из меню и кода), с богатым визуальным языком (ghost-миниатюры, превью зон, живой reorder-превью полосы).
- **Персистентность**: JSON со схемой, `Apply` с отчётом об исправлениях, «спящие» состояния незарегистрированного контента, сброс раскладки без потери документов.
- **Композиция**: fluent-builder в ядре и AXAML-определения (`BerthWorkspace.Definition`) — декларативная сборка без кода.

Поведение выверено построчной сверкой с открытыми исходниками IntelliJ IDEA и закреплено спецификациями, где каждый нормативный пункт покрыт тестом с тем же id.

## Пакеты

- **Berth.Core** — модель раскладки и операции; без зависимостей на UI-фреймворк, пространство имён `Berth`.
- **Berth.Avalonia** — контролы, материализующие модель; пространство имён `Berth.Controls`.

```sh
dotnet add package Berth.Avalonia   # ядро приходит транзитивно
```

Минимальный пример и проводка — в [docs/getting-started.md](docs/getting-started.md).

# Berth

**Библиотека докинга для [Avalonia](https://avaloniaui.net) — по строгой модели IntelliJ IDEA.**

Шесть фиксированных слотов служебных панелей, документная зона — дерево разбиений
с группами вкладок, плавающие окна, сохранение и восстановление раскладки.
Сознательно **не** произвольная компоновка: строгие правила → перечислимое
пространство состояний → надёжность.

## Модель одним взглядом

```text
┌──┬─────────────────────────────────┬──┐
│п │ ┌───────┬───────────┬─────────┐ │п │
│о │ │ левая │ док-зона  │ правая  │ │о │
│л │ │ панель│(документы)│ панель  │ │л │
│о │ ├───────┴───────────┴─────────┤ │о │
│с │ │        нижняя панель        │ │с │
│а │ └─────────────────────────────┘ │а │
└──┴─────────────────────────────────┴──┘
```

Три стороны × две группы = **шесть слотов** служебных панелей (tool windows; в
спецификациях — «тул-панели»); у каждой панели **пять режимов отображения**: dock pinned,
dock unpinned, undock (поверх раскладки, не занимая места), float (окно поверх главного),
window (независимое окно). Документная зона — **n-арное дерево разбиений** с группами
вкладок; вкладки панелей и документы живут в одной модели дерева и переносятся одними и
теми же командами.

## Почему не произвольная компоновка

Произвольная компоновка (free-form docking) разрешает любую вложенность контейнеров — и
вместе с ней получает неограниченное пространство состояний и неисчислимые краевые случаи:
хронические болезни жанра. Berth выбирает противоположный полюс — ограниченную модель
IntelliJ IDEA:

- **Перечислимость по построению.** Панели существуют только в шести слотах; произвольная
  вложенность допустима лишь в документной зоне — и то в виде дерева, приводимого к
  канонической форме. Инварианты закреплены property-тестами.
- **Спецификация как истина.** Поведение определяют спецификации: каждый нормативный пункт
  имеет стабильный номер (`TW-…`, `DA-…`, `INV-…`) и покрыт тестом с тем же номером.
  Спорные места выверены построчной сверкой с открытыми исходниками intellij-community.
- **Команды, а не жесты.** Каждое завершённое перетаскивание сводится к команде ядра,
  доступной также из меню и кода; до отпускания кнопки жест — чистая отрисовка, поэтому
  отмена ничего не стоит и не оставляет следов в состоянии.

## Возможности

- **Служебные панели** — шесть слотов, пять режимов с вытеснением по слоям, автоскрытие
  при потере фокуса, полосы иконок с быстрым доступом «⋯», двухуровневые меню, сочетания
  клавиш активации; геометрия сторон и пар — в долях, пиксели в ядро не проникают.
- **Документная зона** — n-арное дерево разбиений с группами вкладок и приведением к
  канонической форме, отдельные окна документов; единая модель дерева с содержимым
  панелей — вкладка переезжает между панелью и документной зоной теми же командами.
- **Плавающий слой** — настоящие окна ОС на десктопе, псевдоокна поверх рабочей области в
  браузере; сохранённый режим не подменяется — упрощается только представление.
- **Перетаскивание** — иконки, заголовки и вкладки, внутри окна и между окнами — тонкий
  слой над командами ядра, с продуманным визуальным языком: миниатюры содержимого у
  курсора, предпросмотр зон вставки, живая перестановка вкладок в полосе.
- **Сохранение раскладки** — версионированный JSON, применение с отчётом об исправлениях,
  «спящие» состояния незарегистрированного содержимого, сброс раскладки без потери
  открытых документов.
- **Композиция** — программный построитель в ядре и декларативные AXAML-определения
  (`BerthWorkspace.Definition`) — сборка рабочей области без кода.

## Быстрый старт

```sh
dotnet add package Berth.Avalonia
```

Пакета два: **Berth.Core** — модель раскладки и операции, без зависимостей от
UI-фреймворка (пространство имён `Berth`); **Berth.Avalonia** — элементы управления,
материализующие модель (пространство имён `Berth.Controls`). Установка второго приносит
ядро как зависимость.

Композиция собирается построителем и передаётся `BerthWorkspace`:

```csharp
var composition = new LayoutCompositionBuilder()
    .AddToolWindow("project", "Project", w => w
        .Slot(ToolWindowSide.Left, ToolWindowGroup.Primary)
        .Content(_ => new TextBlock { Text = "Project tree" }))
    .AddDockContent(
        id => id.StartsWith("doc:", StringComparison.Ordinal),
        id => new TextBox { Text = $"// {id[4..]}", AcceptsReturn = true })
    .Open("project")
    .OpenDocument("doc:README.md")
    .Build();

var workspace = new BerthWorkspace
{
    Registry = composition.Registry,
    Lifecycle = composition.Lifecycle,
    State = composition.State, // привязка двусторонняя: жесты записывают результат команд обратно
};
```

Каждый жест пользователя — команда ядра; результат присваивается обратно в `State`, и
приложение наблюдает изменения через привязку или
`GetObservable(BerthWorkspace.StateProperty)`.

<details>
<summary>Та же рабочая область декларативно — в AXAML</summary>

```xml
<berth:BerthWorkspace xmlns:berth="using:Berth.Controls">
  <berth:BerthWorkspace.Definition>
    <berth:BerthLayoutDefinition>
      <berth:ToolWindowDefinition Id="project" Title="Project"
                                  Side="Left" Group="Primary" IsOpen="True">
        <views:ProjectView/>
      </berth:ToolWindowDefinition>
      <berth:DockContentDefinition TabIdPrefix="doc:">
        <berth:DockContentDefinition.ContentTemplate>
          <DataTemplate><views:DocumentView/></DataTemplate>
        </berth:DockContentDefinition.ContentTemplate>
      </berth:DockContentDefinition>
      <berth:DocumentDefinition Id="doc:README.md"/>
    </berth:BerthLayoutDefinition>
  </berth:BerthWorkspace.Definition>
</berth:BerthWorkspace>
```

</details>

Сохранение раскладки — политика приложения поверх кирпичиков ядра: `LayoutPersistence`
(JSON со `SchemaVersion`), `Apply` с отчётом об исправлениях, `ResetToDefaults`, проверка
сохранённых границ окон. Полное руководство — [docs/getting-started.md](docs/getting-started.md).

## Платформы

| Платформа | Поддержка | Плавающий слой |
|---|---|---|
| Windows / macOS / Linux | полная | настоящие окна ОС: float поверх главного и независимые window |
| Браузер (WebAssembly) | полная | псевдоокна поверх рабочей области |
| Android / iOS | сборка по остаточному принципу | не материализуется; поведение на сенсорном вводе не специфицируется |

Сохранённый режим панели никогда не подменяется: на платформе без нужных возможностей
упрощается только представление (`Window` → `Float` → `Undock`,
[ADR-0006](docs/adr/0006-platform-capabilities.md)) — вернувшись на полноценную платформу,
раскладка ведёт себя как прежде.

## Демонстрация

`samples/Berth.Demo` — «мини-IDE»: панели двух путей поставки содержимого (MVVM и фабрики
элементов управления), документы, вкладки терминала, сочетания клавиш, сброс раскладки и
отложенное автосохранение.

```sh
dotnet run --project samples/Berth.Demo.Desktop   # десктоп: сохранение в файл, настоящие окна

dotnet workload install wasm-tools                # разово, для браузерного хоста
dotnet run --project samples/Berth.Demo.Browser   # браузер: http://localhost:5235, localStorage
```

## Архитектура и метод

- **Ядро без UI.** `Berth.Core` не ссылается на Avalonia — граница закреплена
  архитектурным тестом. Вся логика раскладки — чистые переходы неизменяемого состояния;
  слой интерфейса лишь материализует состояние и переводит ввод в команды
  ([ADR-0002](docs/adr/0002-core-avalonia-layering.md)).
- **Раскладка хранит только идентификаторы.** Содержимое создают фабрики приложения —
  ленивое создание, «спящие» состояния, устойчивое восстановление
  ([ADR-0003](docs/adr/0003-layout-content-separation.md)).
- **Перетаскивание поверх команд.** Библиотека функционально полна и без него: всё
  достижимо из меню и кода ([ADR-0004](docs/adr/0004-dnd-over-commands.md)).
- **Пиксели — на границе интерфейса.** В ядре только веса и доли; экранные координаты
  проверяются при восстановлении.

Архитектурные решения записаны в [docs/adr/](docs/adr/).

## Репозиторий

Решение — `Berth.slnx`: библиотека в `src/`, тесты — модульные и property-тесты ядра,
headless-тесты интерфейса, интеграционные тесты демо — в `tests/`, демонстрационные
хосты — в `samples/`. CI проверяет форматирование, выполняет все три набора тестов,
собирает десктопное демо и упаковывает пакеты NuGet; предупреждения сборки считаются
ошибками. Папка репозитория пока носит историческое имя `IdeaDocking`.

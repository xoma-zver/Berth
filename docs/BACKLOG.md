# Бэклог

Правила:
- Задача = один ревьюируемый дифф (ветка → ревью → мердж в master).
- Сессия разработки берёт верхнюю незакрытую задачу текущей фазы, если владелец не указал иное.
- Завершение задачи включает: тесты на затронутые пункты спеки, обновление этого файла, ноль предупреждений сборки.

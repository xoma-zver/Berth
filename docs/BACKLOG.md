# Бэклог

Правила:
- Задача = один ревьюируемый дифф (ветка → ревью → мердж в main).
- Сессия разработки берёт верхнюю незакрытую задачу, если владелец не указал иную.
- Завершение задачи включает: тесты на затронутые пункты спеки, обновление этого файла, ноль предупреждений сборки.

## Статус v1

Фазы 0–7 завершены (каркас; ядро tool windows; статический UI и ходячий скелет; режимы и
фокус; документная зона UI; DnD внутри окна; плавающие окна; интеграция и выпуск 0.1.0),
как и разовые хозяйственные задачи. Летопись выполненных задач — с решениями владельца,
находками реализации и итогами ревью по каждой — хранится в git-истории этого файла
(версии до 2026-07).

## Хозяйственное (вне фаз, по решению владельца)

- [x] Нереентерабельность прохода проекции — TW-9.14 (2026-07, гипотеза владельца об «окне-призраке», подтверждена регрессионным тестом): команда, порождённая внутри прохода `Sync` (реакции на перенос фокуса при переприсоединении хостов DA-6.4/TW-6.1, события платформенных окон TW-7.x), влетала вложенным проходом в середину подпроходов; согласование окон документов по пересечению вкладок (DA-1.3) заставало кэш `_documents` очищенным и создавало окно-дубликат — полосу вкладок без контента, гибнущую на следующем проходе. Страж в `BerthWorkspace.Sync`: вложенный запрос помечает проекцию устаревшей, перепроход по актуальному состоянию — по завершении текущего. По ревью владельца: сходимость ограничена пределом перепроходов (осциллирующая пара команд — громкое исключение вместо молчаливого зависания UI-потока; дедупликация равных состояний не исключает двухтактный цикл), а полный сброс кэшей при смене `Registry`/`Lifecycle` отведён в ту же воронку — немедленный `Reset()` из середины прохода ронял внешний проход на ближайшем обращении к кэшу. Спека: TW-9.14, кросс-ссылка в DA-9.6; тесты `ProjectionReentrancyTests` (оба слоя плавающих окон, детерминированный шов — действие из `OnAttachedToVisualTree` контента, с фиксацией срабатывания именно внутри прохода): призрак, осцилляция, сброс из середины прохода. Вторым коммитом (вопрос владельца) — предел глубины вложенности команд: фокусные реакции рекурсивны по стеку мимо проекции (каждый уровень запускает свой ограниченный проход), конечность держалась только на дисциплине проводки, а регрессия умерла бы неперехватываемым переполнением стека; превышение предела — громкое исключение, тест — фокусный пинг-понг тел двух панелей (`Open(a)` ↔ `Open(b)` из `OnGotFocus`)

# ADR-0005. Документная зона — n-арное дерево сплитов

Статус: принято (2026-07).

## Контекст

«Произвольная грид-сетка» документов должна допускать сплиты в обе стороны, перенос вкладок и вытаскивание в отдельные окна, оставаясь сериализуемой и проверяемой.

## Решение

Модель — дерево: узел-сплит (ориентация + упорядоченные дети + доли) и лист-«таб-группа» (упорядоченные `Id` документов + активный). N-арное, как grid в VS Code (а не бинарное, как в IDEA): меньше глубина, естественные тройные сплиты.

После каждой операции — нормализация: сплит с единственным ребёнком схлопывается; вложенный сплит той же ориентации вливается в родителя; пустые таб-группы удаляются (кроме последней корневой).

Отдельное окно с документами = собственные bounds + такое же дерево. Рекурсия — в данных, не в иерархии контролов.

## Следствия

- Инварианты («дерево нормализовано», «каждый документ ровно один раз», «доли в сумме 1») — идеальная мишень property-тестов.
- Любая достижимая конфигурация выражается деревом — «грид» не нужен как отдельная сущность.
- Детальная спека поведения — отдельным документом (задача 1.4 бэклога).

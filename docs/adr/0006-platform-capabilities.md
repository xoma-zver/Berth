# ADR-0006. Платформенные возможности и плавающие окна

Статус: принято (2026-07).

## Контекст

На десктопе Avalonia даёт настоящие окна; в браузере приложение — один canvas в одном TopLevel, окон нет. Требование — работать и там, и там.

## Решение

Хостинг плавающего контента — за абстракцией (реальное `Window` на десктопе; псевдоокно в оверлей-слое приложения в браузере: свой z-порядок, перемещение, ресайз). Платформа объявляет возможности: `CanFloat`, `CanUseWindowed`.

Хранимый `Mode` панели никогда не деградирует — деградирует только *эффективный* режим отображения: `Window` → `Float` → `Undock` по мере отсутствия возможностей. Раскладка, перенесённая на «полную» платформу, восстанавливает исходное поведение.

Мультиоконный WASM (отдельные browser-окна со вторым канвасом и синхронизацией) сознательно вне рамок v1.

## Следствия

- Ядро и сериализация полностью платформонезависимы; ветвление — в одном месте UI-слоя.
- Меню режимов строится по возможностям платформы (недоступное скрыто).
- Оверлей-псевдоокна — собственный мини-window-manager в UI-слое; тестируется headless.

# ADR-0002. Слои: ядро без Avalonia + тонкий UI-слой

Статус: принято (2026-07).

## Контекст

Требование максимальной надёжности и тестируемости; UI-фреймворк — самая нестабильная и трудно тестируемая часть стека.

## Решение

- `src/IdeaDocking.Core` — модель раскладки, все операции (валидируемые переходы состояния), инварианты, сериализация. Ни одной ссылки на Avalonia. Граница закреплена архитектурным тестом (`ArchitectureTests`).
- `src/IdeaDocking.Avalonia` — материализация состояния в контролы, ввод (клики, фокус, DnD), хосты плавающих окон. Ссылается на Core, обратной зависимости нет.
- Вся логика «что произойдёт» живёт в Core; UI отвечает только за «как выглядит и как вводится».

## Следствия

- 90 % поведения тестируется обычным xUnit + property-тестами без UI; UI-слой — через Avalonia.Headless.
- Пиксельные величины в ядро не попадают: только веса/доли (пиксели — на границе UI).
- Дублирование терминов между слоями (состояние ↔ контролы) — осознанная цена.

# ADR-0003. Раскладка хранит только идентификаторы; контент — за фабриками

Статус: принято (2026-07).

## Контекст

Раскладка должна переживать перезапуск, отсутствующие плагины и эволюцию приложения; сериализовать сами контролы/вью-модели — путь к хрупкости.

## Решение

Состояние раскладки оперирует только строковыми `Id` и геометрией. Контент создают зарегистрированные приложением дескрипторы с фабриками (аналог `ToolWindowFactory` в IDEA и view descriptors в VS Code). Создание ленивое по умолчанию; политики жизненного цикла — по осям «создание» и «удержание» (спека, раздел 9).

Сохранённые состояния незарегистрированных `Id` не выбрасываются, а «спят» и сериализуются обратно (round-trip), пока не появится регистрация.

## Следствия

- Восстановление устойчиво по построению: неизвестное — спит, новое — встаёт в дефолт.
- Персистентность контента (что внутри документа) — отдельный контракт приложения, не забота раскладки.
- Любая панель/документ обязаны иметь стабильный `Id`; анонимного контента не существует.

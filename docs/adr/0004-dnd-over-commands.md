# ADR-0004. Drag-and-drop — тонкий слой над командами ядра

Статус: принято (2026-07).

## Контекст

DnD — главный источник сложности и багов докинг-библиотек, при этом функционально он дублирует операции, доступные из меню.

## Решение

DnD не умеет ничего, чего не умеет команда ядра. Каждый завершённый жест сводится к вызову той же операции (`Move`, `SetMode`, перенос вкладки…), что доступна из меню и из кода. Жест до отпускания — чистая визуализация (ghost, подсветка целей), не меняющая состояние.

Реализация — pointer capture + собственный контроллер перетаскивания (не OS drag-and-drop): единообразно работает на десктопе и в браузере; между окнами на десктопе — экранные координаты + хит-тест окон приложения.

## Следствия

- Библиотека функционально полна без DnD (всё доступно через меню/клики) — DnD добавляется поздней фазой без риска для ядра.
- Тестировать нужно только маппинг «жест → команда», а не «жест → мутация состояния».
- Отмена жеста (Esc/отпускание вне целей) тривиальна: команда просто не вызывается.

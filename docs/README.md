# Документация

Карта:

- [spec/](spec/) — поведенческие спецификации. Истина о том, «что должно происходить»; каждый нормативный пункт имеет стабильный id и покрывается тестом с тем же id.
  - [tool-windows.md](spec/tool-windows.md) — тул-панели (слоты, режимы, автоскрытие, персистентность).
  - document-area.md — документная зона (дерево сплитов) — будет написана в задаче 1.4.
- [adr/](adr/) — архитектурные решения. Короткие, нумерованные, не переписываются задним числом (новое решение = новый ADR со ссылкой на отменяемый).
- [BACKLOG.md](BACKLOG.md) — фазы и задачи. Сессия разработки начинается отсюда.
- [OPEN-QUESTIONS.md](OPEN-QUESTIONS.md) — вопросы, требующие решения владельца проекта. Всё, в чём исполнитель не уверен, попадает сюда, а не решается молча.

Порядок изменения поведения: сначала правка спеки (или новый пункт), затем код и тест с id пункта.

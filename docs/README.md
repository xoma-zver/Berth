# Документация

Карта:

- [getting-started.md](getting-started.md) — прикладной гайд: установка пакетов, композиция раскладки (builder и AXAML), проводка `BerthWorkspace`, персистентность.
- [spec/](spec/) — поведенческие спецификации. Истина о том, «что должно происходить»; каждый нормативный пункт имеет стабильный id и покрывается тестом с тем же id.
  - [tool-windows.md](spec/tool-windows.md) — тул-панели (слоты, режимы, автоскрытие, персистентность).
  - [document-area.md](spec/document-area.md) — документная зона: дерево таб-групп, окна документов, общая модель дерева для контента панелей.
- [adr/](adr/) — архитектурные решения. Короткие, нумерованные, не переписываются задним числом (новое решение = новый ADR со ссылкой на отменяемый).
- [reference/](reference/) — справочные материалы (не нормативные): как устроен докинг у эталона (IntelliJ IDEA) и в чём Berth сознательно расходится.
  - [idea-tracing.md](reference/idea-tracing.md) — тул-панели эталона: ключевые файлы, модель соответствия терминов, механизмы по подсистемам, расхождения.
  - [idea-tracing-document-area.md](reference/idea-tracing-document-area.md) — документная зона эталона: дерево сплитов, активность, окна документов, мост «панель ↔ редакторы».
  - Построчная трассировка каждого пункта спек к исходникам эталона, по которой принимались решения, — в git-истории этих файлов.
- [BACKLOG.md](BACKLOG.md) — задачи. Сессия разработки начинается отсюда; летопись выполненного — в git-истории файла.
- [OPEN-QUESTIONS.md](OPEN-QUESTIONS.md) — вопросы, требующие решения владельца проекта. Всё, в чём исполнитель не уверен, попадает сюда, а не решается молча.

Порядок изменения поведения: сначала правка спеки (или новый пункт), затем код и тест с id пункта.

# Открытые вопросы

Вопросы, требующие решения владельца проекта. Формат: контекст → варианты → рекомендация исполнителя.

Жизненный цикл: после решения владельца суть переносится в спеку/ADR/бэклог, а вопрос **удаляется** — история решения остаётся в git (коммит ссылается на номер вопроса). Нумерация сквозная и не переиспользуется: вопросы №1–№6 закрыты (имя Berth, платформы, язык, флаги спеки, дерево в панелях, семантика Undock); следующий вопрос получает №7.

## №7 — Терминология русской документации: кальки с английского

**Контекст.** Владелец отметил (2026-07), что кальки режут глаз («тул-панели», «free-form», «воркспейс»). Витрина уже переведена на русские термины: `README.ru.md` использует «служебные панели» (tool windows), «произвольная компоновка» (free-form docking), «рабочая область» (workspace), «полосы иконок» (stripes), «содержимое» (контент), «сохранение раскладки» (персистентность). Спецификации, справочники и getting-started продолжают использовать «тул-панель», «стрип», «сплит», «контент», «персистентность» как определённые термины — сотни вхождений; получился двойной словарь.

**Варианты.**
1. Сквозная замена терминов во всей `docs/` (спеки, ADR, справочники, руководство) — единый словарь; редакционный дифф по принятым спекам большой, но нормативные id (`TW-…`, `DA-…`) не затрагиваются.
2. Оставить спецификациям их словарь как «технический язык проекта», витрине — русский; закрепить таблицу соответствия терминов в `docs/README.md`.
3. Менять постепенно — по мере правок конкретных документов.

**Рекомендация исполнителя.** Вариант 1 отдельной сессией-уборкой: замена чисто редакционная, а двойной словарь — постоянный источник трения при чтении и написании текстов. До решения новые тексты писать русскими терминами из `README.ru.md`.

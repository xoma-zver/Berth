## 5. Дерево таб-групп внутри тул-панели — ОТКРЫТ (вариант «в» на согласовании)

Наблюдение владельца (терминал Rider): внутри панели — полноценное дерево таб-групп со Split Right/Down, переносом вкладок в редактор и обратно, вытаскиванием в отдельное окно. Подтверждено документацией (Reworked Terminal, дефолт с 2025.2); реализовано в плагине терминала (каталоги `arrangement/`, `vfs/`), а не как общая услуга платформы — Run/Find живут на плоских вкладках ContentManager.

Предлагаемая модель (в): панель может хостить **то же дерево таб-групп, что и докзона** (ADR-0005), с правилами изоляции:
- слой слотов видит панель как атом: по сайдбарам панель перемещается только целиком, вместе со своим деревом;
- вкладка панели перемещается внутри своей панели или в докзону; из докзоны — обратно только в свою панель-владельца; владелец определяется реестром по id вкладки, само дерево хранит только id — схема сериализации не меняется;
- документ не может попасть в панель; вкладка чужой панели — тоже; панель внутри панели — невыразима.

Вырожденный случай (одна группа, одна вкладка) = обычная панель, полоса табов скрыта. Персистентность структуры дерева — часть раскладки; содержимое вкладок — фабрики приложения (ADR-0003). Плоские вкладки (Run-стиль) — дерево из одной группы.
